我該學哪種程式語言呢?

我該學哪種程式語言呢?

作者：陳鍾誠

摘要



剛進入程式設計領域的人最常問的問題之一是 “我該學哪一種語言好呢”。這個問題見仁見智，以目前的眼光來看， 大致上可以分為三個方向，分別是Microsoft, Linux, Java，我們將其表列如下。
1. Microsoft
Visual Basic 視窗介面簡單易用，但能力不強。
ASP 網頁設計簡單易用，但能力不強。
C# 類似 Java, 但尚未普及，不能跨平台。
C 以MFC 函式庫為主，很難學。
2. Linux
以 C 為主，網頁設計為 PHP，但通常是Linux系統玩家使用。
3. Java
視窗介面, 網頁設計, 資料庫, … 都很好用，只要會 Java 一套就好了，跨平台效果很好。

對一個非Linux 玩家，且不想常常跟著 Microsoft 學習新的微軟語言的人而言，Java 會是一個不錯的選擇，其好處是：
1. 免費 不需購買軟體也不用盜版，就可以免費使用，直接上 java.sun.com 去下載就行了。
2. 跨平台 Windows 與 Linux 上都可以用。
3. 資源豐富 網路上到處都可找到簡介，免費的工具，相關書籍很多。
4. 語言強大 Java 語言比 VB 強大，函式庫比C, C++ 多很多。

其缺點是主要是
1. 安裝較麻煩 由於授權的問題，Java 常常要安裝很多個系統，例如核心函式庫要從 java.sun.com 下載，整合環境可裝 JCreater LE, 寫網站時要下載 Tomcat 等。

由於優點很多，缺點相對上不嚴重，因此、Java 相當值得推薦。

但是若你想學習特定的主題，例如 專業的電動設計，Linux作業系統的核心等，則不應該使用 Java，因為其虛擬機器的架構使其執行較慢，跟不上 C,C++ 語言的速度，然而、 C,C++ 語言的問題是標準的函式庫太小，網路、資料庫、介面等功能都不支援，除非你進入的特定領域已經累積了大量的函式庫供你使用，否則 C 語言將無用武之地，另外、C,C++ 語言由於指標的濫用，常有很多 bug ，也是 C, C++ 的缺點之一。

然而、對於 Linux 玩家而言， C/C++ 當然是首選，Linux 已經累積了大量的函式庫可以用，其修改系統的能力則是 Java 所望塵莫及的，因此若你的目標是製作 Linux 的系統程式， C/C++ 當然是首選。

然而、如果你想開發 Windows 系統的的程式，例如 Windows 的 USB 介面驅動程式等，則除了 CC++ 以外，還要熟悉 MFC 等函式庫，但是相當難學就是了。
